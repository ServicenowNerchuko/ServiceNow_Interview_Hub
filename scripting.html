2. Can we abort a record insertion/update using Business Rule?

Yes, Current.setAbortAction(true);

3. Difference between Client Script and Business Rule?

Client Script runs on the browser side and is used for front-end form behavior, whereas Business Rule runs on the server side and is used for backend processing and database operations.

Real-Time Scenario Answer :

For example, if we want to make Assignment Group mandatory when Priority is High, we use Client Script because it is a UI behavior.

If we want to automatically create a task, update another table, or trigger notifications after Incident creation, we use Business Rule because it requires server-side processing.

4. How do you differentiate between incident and problem?

An Incident is an unplanned interruption to an IT Service or reduction in quality of an IT Service where as Problem is the underlying cause of one or more incidents.

5. What is Business Rules?

Business Rules are Server-Side Scripts that execute when database operations occur and update related records.

They run at different stages like Before, After, Async& Display.

6.How do you integrate Servicenow with other systems?

We integrate ServiceNow with external systems mainly using REST APIs, SOAP APIs, IntegrationHub, and MID Servers based on business requirements. In most real-time projects, REST API is preferred because it is lightweight, fast, and easy to maintain.

The integration process includes authentication, payload processing, data transformation, error handling, and automatic record updates. For secure on-premise integrations, MID Server acts as a bridge between ServiceNow and internal systems.

7.What happens if you use current.update() inside a business rule?

Technically, current.update() can be used in After Business Rules because the record is already inserted or updated in the database at that stage.

However, it is generally avoided because it may cause recursive execution and performance issues. In most cases, Before Business Rules are preferred since changes made to current are automatically saved without using current.update().
